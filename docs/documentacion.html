DiveComputer — Documentación

Ordenador de buceo de código abierto basado en ESP32. Muestra rumbo compensado por inclinación, profundidad y tiempo de inmersión en un display TFT de 1.8". Incluye portal WiFi para calibración del magnetómetro sin necesidad de herramientas adicionales.

Alimentación

El ESP32-C3-DevKitC-02 se alimenta por USB-C a 5 V y dispone de un regulador interno que genera 3.3 V para todos los periféricos. No se necesita regulador externo.

⚠ Tensiones lógicas
El ESP32-C3 opera a 3.3 V. El ST7735, HMC5883L y LSM303 también funcionan a 3.3 V. No conectes directamente señales de 5 V a ninguno de estos pines.

Compilar y flashear

Requisitos

  • PlatformIO (extensión VS Code recomendada)
  • Driver USB-CDC — en Windows puede requerir reinicio tras conectar por primera vez
  • Python 3.8+ (gestionado automáticamente por PlatformIO)

Comandos habituales

# Compilar
pio run

# Compilar y flashear (ESP32-C3, entorno por defecto)
pio run --target upload

# Flashear entorno ESP32-S3
pio run -e esp32-s3-devkitc-1 --target upload

# Abrir monitor serie (115 200 baud)
pio device monitor

# Flash + monitor en un solo comando
pio run --target upload && pio device monitor

# Limpiar artefactos de compilación
pio run --target clean
ℹ USB CDC
El firmware habilita ARDUINO_USB_CDC_ON_BOOT=1. El monitor serie aparece como puerto virtual tras la enumeración USB; si no aparece, desconecta y reconecta el cable.

Arquitectura del firmware

Todo el código reside en src/main.cpp. No hay módulos separados.

Orden de inicialización (setup())

  1. Serial — USB CDC a 115 200 baud. Espera hasta 2 s a la enumeración USB.
  2. TFT — SPI.begin(9,−1,8) → tft.initR(INITR_BLACKTAB) → rotación landscape → splash screen.
  3. I²C — Wire.begin(I2C_SDA, I2C_SCL) con los pines definidos en platformio.ini.
  4. HMC5883L — mag.begin(). Bucle infinito si el sensor no responde.
  5. LSM303 — accel.begin(0x18). SA0=GND → dirección 0x18.
  6. Preferencias — Carga calibración almacenada en flash NVS (namespace "compass").
  7. WiFi AP — Punto de acceso Brujula-Calib en canal 1 + servidor HTTP en puerto 80.

Bucle principal (loop() — cada 30 ms)

server.handleClient();            // procesa peticiones HTTP del portal

accel.getEvent(&accEv);            // leer acelerómetro
mag.getEvent(&magEv);              // leer magnetómetro

if (calState.collecting)
    updateCalibrationSamples(magEv); // acumular min/max hard-iron

float heading = calculateHeading(magEv, accEv);
g_heading = heading;              // disponible para el portal vía /status

drawHeadingStrip(heading);        // redibuja si cambió ≥1°
drawDepth(g_depth);              // redibuja si cambió ≥0.1 m
drawDiveData(g_tempC, elapsed);   // redibuja si cambió el segundo
if (hdgChanged) drawCompassBar(heading);

delay(30);

Algoritmo de rumbo compensado

La función calculateHeading() calcula el rumbo magnético corregiendo la inclinación de la placa mediante los datos del acelerómetro.

Remapeo de ejes (montaje físico)

El sensor está montado con X hacia abajo, Y hacia la izquierda, Z hacia adelante. Se remapea al marco de la fórmula (X=adelante, Y=izquierda, Z=arriba): 

// Acelerómetro → marco fórmula
ax = -accEvent.acceleration.z;
ay =  accEvent.acceleration.y;
az = -accEvent.acceleration.x;

// Magnetómetro + calibración → marco fórmula
mx = -(magEvent.magnetic.z - cal.oz) * cal.sz;
my =  (magEvent.magnetic.y - cal.oy) * cal.sy;
mz = -(magEvent.magnetic.x - cal.ox) * cal.sx;

Compensación de inclinación (tilt compensation)

// Pitch y roll del acelerómetro
pitch = atan2(-ax, sqrt(ay*ay + az*az));
roll  = atan2(ay, az);

// Componentes horizontales del campo magnético
Xh = mx*cos(pitch) + mz*sin(pitch);
Yh = mx*sin(roll)*sin(pitch) + my*cos(roll) - mz*sin(roll)*cos(pitch);

// Rumbo + declinación magnética (Alcalá la Real ≈ −0.63°)
heading = atan2(Yh, Xh) + (-0.011);   // rad
heading = fmod(heading + 2*PI, 2*PI) * 180.0 / PI;  // → 0..360°
ℹ Declinación magnética
El valor −0.011 rad (≈ −0.63°) está ajustado para Alcalá la Real, España. Para otra ubicación, calcula tu valor en NOAA Magnetic Field Calculator y actualiza la constante declinationAngle en calculateHeading().

Portal de calibración WiFi

Tras arrancar, el ESP32 levanta un punto de acceso WiFi abierto. Conéctate desde cualquier dispositivo y abre el portal para calibrar el magnetómetro sin desmontar ni reconectar nada.

Procedimiento de calibración

  1. Conecta al punto de acceso — busca Brujula-Calib en la lista WiFi de tu móvil y conéctate.
  2. Abre el portal — navega a http://192.168.4.1 en el navegador. Verás el rumbo actual en tiempo real.
  3. Inicia la calibración — pulsa "Iniciar calibración". El estado cambia a calibrating.
  4. Mueve el dispositivo — dibuja un "8" en el aire durante 20–30 s, cubriendo todas las orientaciones posibles (inclina, gira, voltea). Observa cómo aumentan las muestras y los valores min/max.
  5. Detén y guarda — pulsa "Detener y guardar". El firmware calcula offsets hard-iron y escalas soft-iron automáticamente y los almacena en flash NVS.
✓ ¿Cuántas muestras son suficientes?
El firmware solo acepta la calibración si se han recogido más de 50 muestras. Para resultados óptimos, apunta a 200–500 muestras rotando en múltiples ejes.

Algoritmo de calibración hard-iron + soft-iron

// Hard-iron: offset = centro del elipsoide
cal.ox = (maxX + minX) / 2;
cal.oy = (maxY + minY) / 2;
cal.oz = (maxZ + minZ) / 2;

// Soft-iron: escalar cada eje al radio medio
cx  = (maxX - minX) / 2;
avg = (cx + cy + cz) / 3;
cal.sx = avg / cx;   // idem para sy, sz
